# RAG Protocol

This protocol records the public, reviewable boundary for retrieval-augmented
generation used in the PersonaAI paper workflow.

## Purpose

The RAG component is used to assemble relevant public context for
agent-assisted hypothesis evaluation and documentation. It is not used as the
source of reported numerical estimates. Statistical results must come from
deterministic analysis scripts and documented input data.

## Public Boundary

The public repository may include:

- RAG orchestration source code.
- Retrieval tool wrappers.
- Public prompt templates approved for release.
- Synthetic or minimal public examples.
- Tests that do not require private corpora or credentials.
- Corpus provenance manifests.

The public repository must not include:

- Raw private retrieval corpora.
- Generated vector stores or embeddings.
- API keys, access tokens, `.env` files, or service credentials.
- Personal client configuration.
- Internal operational prompts that are not approved for release.

## Reproducibility Requirements

Every public RAG workflow should document:

- The retrieval corpus name and version.
- Public access route or restriction reason.
- Indexing method at a high level.
- Query or retrieval policy relevant to manuscript claims.
- Whether generated text was used for hypothesis organization,
  documentation, or result interpretation.
- Which downstream numerical outputs were generated by deterministic scripts.

## Manuscript Interpretation

RAG-assisted outputs should be described as context assembly and workflow
support. They should not be presented as independent statistical evidence unless
the supporting calculation is implemented in deterministic code and included in
the relevant analysis directory.
