# Agent Assistance and Reproducibility Disclosure

This repository separates agent-assisted workflow guidance from deterministic
statistical analysis.

## What Was Agent-Assisted

- Structuring candidate aging hypotheses into testable genetic questions.
- Selecting feasible evidence layers for public-data analysis.
- Mapping analysis goals to reusable statistical genetics modules.
- Identifying interpretation gaps and targeted sensitivity analyses.
- Drafting manuscript-oriented summaries from executed outputs.

## What Was Deterministic

- pQTL locus extraction.
- Allele harmonization.
- Lead-variant Wald ratio Mendelian randomization.
- Bayesian colocalization with `coloc.abf()`.
- Summary table and figure generation.

All reported numerical estimates are generated by scripts in
`analysis/creld2_human_genetics/scripts/` and are summarized in
`analysis/creld2_human_genetics/results/`.

## What Is Not Claimed

- The workflow does not establish definitive causal inference from a single
  pQTL instrument.
- The workflow does not treat all aging outcomes as equivalent validation
  cohorts.
- The workflow does not rely on an unpublished model API for numerical
  estimates.
- The public protocol cards are generalized documentation of the analysis
  logic and do not expose internal operational prompts or private credentials.

## Single-Cell Aging-Atlas Workflow

### What Was Agent-Assisted (Single-Cell)

- Structuring the biological question into competing cell-of-origin and niche
  hypotheses (CIRBP cell-of-origin in liver, PGAT sex-specific vascular
  endothelial cell niche).
- Assessing feasibility against available tissue, gene, and metadata coverage.
- Mapping analysis goals to MCP analysis modules: data extraction, single-cell
  differential expression and lineage analysis, and cell-cell interaction.
- Gap-oriented cross-checking of verdict-driving results.
- Drafting verdict summaries from executed outputs.

The agent orchestrates the workflow (deciding stages, parameters, and sequence,
and interpreting results) while the MCP engines and deterministic scripts
execute the analysis.

### What Was Deterministic / Independently Verified

- A local `scanpy`/`scipy` cross-check independently recomputes the
  verdict-driving metrics and reports them against MCP outputs and reference
  values (a 3-way comparison with honest match labels).

### MCP-First Note

Primary analysis runs through the bundled MCP engines in
`personaai/scrna_mcp/`. The local cross-check is the transparency and
verification layer; the MCP internals are otherwise opaque.

### What Is Not Claimed

- Single-cell association does not establish causal inference.
- Subsampled atlas objects may distort composition. This is why one PGAT
  proportion metric did not reproduce in the cross-check, and it is reported as
  a limitation rather than a confirmed result.
- No numerical estimate depends on an unpublished model API.
